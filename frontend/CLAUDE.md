# frontend/

## Purpose

React/TypeScript dashboard UI for the Oasis simulation.

## Tech

- React 19, TypeScript 5.9 (strict mode)
- Vite 8 (bundler)
- Tailwind CSS 4 (via @tailwindcss/vite plugin)
- openapi-typescript (dev — generates types from simulation API)

## Key Files

- `src/main.tsx` — App entry point, renders `<App />`
- `src/App.tsx` — Root component with view switching and simulation controls
- `src/api/client.ts` — Typed API client using native fetch
- `src/hooks/useSimulation.ts` — Central state management hook (polling + auto-advance)
- `src/types/simulation.ts` — Re-exports generated types under stable names
- `src/contracts/simulation.d.ts` — Auto-generated TypeScript types from OpenAPI (do not edit)
- `src/index.css` — Tailwind imports

## Commands

```bash
pnpm install  # Install dependencies
pnpm dev      # Start dev server (port 5173, proxies API to localhost:8080)
pnpm build    # Production build
```

## API Types

Types are auto-generated from the simulation's OpenAPI schema — never hand-write API types.

- `src/contracts/simulation.d.ts` — generated by `make codegen` (do not edit manually)
- `src/types/simulation.ts` — re-exports generated types under stable names for consumers
- To update after API changes: run `make codegen` from the project root

## Conventions

- Strict TypeScript: no unused locals/parameters, no fallthrough cases
- ES2022 target, ESNext modules
- Base URL: `.` (relative imports from project root)
- Import API types from `src/types/simulation.ts`, not directly from `src/contracts/`
